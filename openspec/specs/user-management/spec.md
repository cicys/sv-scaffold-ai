# 用户管理规范

## 目标

定义当前仓库的用户信息装载、用户资料维护、后台用户管理，以及用户导入导出行为。

## 要求

### 要求：当前登录用户信息装载
客户端装载登录态时必须使用统一的用户信息接口。

#### 场景：加载当前登录用户

- 当客户端调用 `/system/user/getInfo` 时
- 则返回体必须包含 `user`、`permissions`、`roles`
- 并且当当前用户数据不在可访问范围时必须显式失败

### 要求：个人资料维护
当前用户的资料、角色组、岗位组与密码修改必须走统一入口。

#### 场景：查看或更新个人资料

- 当当前用户调用 `/system/user/profile` 时
- 则系统必须返回用户信息、角色组和岗位组
- 并且更新个人资料时必须校验手机号和邮箱唯一性

#### 场景：修改个人密码

- 当当前用户调用 `/system/user/profile/updatePwd` 时
- 则系统必须先验证旧密码正确
- 并且新密码不得与旧密码相同

### 要求：后台用户维护
管理员维护用户时必须遵守唯一性与自保护规则。

#### 场景：新增或修改用户

- 当管理员新增或修改用户时
- 则登录账号、手机号、邮箱必须做唯一性校验
- 并且新增用户时密码必须经过 BCrypt 哈希

#### 场景：删除用户

- 当管理员删除用户时
- 则当前登录用户不得删除自己

### 要求：导入导出能力
用户导入导出必须继续保留当前 Excel 入口。

#### 场景：导出用户

- 当管理员调用 `/system/user/export` 时
- 则系统必须按当前筛选条件导出 Excel

#### 场景：导入用户

- 当管理员调用 `/system/user/importData` 时
- 则系统必须接收 `file` 与 `updateSupport`
- 并且 `/system/user/importTemplate` 必须继续提供模板下载
